# Navidrome 3DS Client

A native homebrew music client for the New Nintendo 3DS that connects to a Navidrome server.

## Project Structure

```
navidrome-3ds/
├── Makefile
├── source/
│   ├── main.c          # Entry point, app loop
│   ├── api.c / api.h   # Navidrome/Subsonic API calls
│   ├── audio.c / audio.h  # DSP audio streaming
│   ├── ui.c / ui.h     # Screen rendering (citro2d)
│   └── config.c / config.h # Server settings storage
├── romfs/
│   └── config.ini      # Default config (editable on SD card)
└── build/              # Generated by make
```

## Requirements

- devkitPro with 3DS libraries: `3ds-dev`, `3ds-mbedtls`, `3ds-curl`
- A running Navidrome server accessible on your local network
- A New 3DS (or Old 3DS, but playback is smoother on New 3DS)
- Custom firmware (Luma3DS) with homebrew enabled

## Building

```bash
make
```

This produces `navidrome.3dsx` in the project root.
Copy it to `/3ds/navidrome/navidrome.3dsx` on your SD card.

## Configuration

Edit `/3ds/navidrome/config.ini` on your SD card:

```ini
[server]
host=192.168.1.100
port=4533
username=admin
password=yourpassword
```

## Controls

| Button | Action |
|--------|--------|
| D-Pad Up/Down | Navigate list |
| A | Select / Play |
| B | Back |
| START | Pause/Resume |
| SELECT | Stop |
| L/R | Volume Down/Up |
| Touch Screen | Tap to select item |

## Architecture Notes

- Uses **Subsonic REST API** (Navidrome is compatible)
- HTTP requests via **libcurl** + **mbedTLS** for HTTPS
- Audio decoded via **libmpg123** or raw PCM streaming
- UI rendered with **citro2d** on both screens
- Config stored on SD card via romfs + sdmc paths
